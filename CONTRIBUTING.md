# Вклад в проект

Спасибо за интерес к проекту! Этот репозиторий использует строгие правила качества и документации.

## Основные требования
- **Коммуникация в чате — на русском**.
- **Код и комментарии в коде — на английском**.
- **Документация — двуязычная**: `docs/ru/` (приоритет) и синхронизированный перевод в `docs/en/`.
- **Conventional Commits**: тип на английском (`feat`, `fix`, `docs`, `refactor`, `test`, `ci`, `build`, `chore`), текст после двоеточия — на русском.
- **CI обязателен**: линтер → форматирование → сборка → тесты.
- **Каждое изменение** должно фиксировать 1–2 идеи в `docs/IDEAS.md` (или другом официальном файле идей).

## Ветки
- Основная ветка: `main` (защищена).
- Рабочие ветки: `feat/<slug>`, `fix/<slug>`, `docs/<slug>`.

## Pull Request
PR должен быть небольшим и фокусным. Используйте шаблон PR и обязательно заполните:
- Summary
- Changes
- Docs
- Changelog
- Test Plan
- Risks
- Rollback
- Ideas
- Instruction

## Документация
Минимальный набор:
- `docs/ru/index.md`, `docs/ru/quickstart.md`, `docs/ru/cli.md`, `docs/ru/config.md`, `docs/ru/troubleshooting.md`
- Синхронизированные версии в `docs/en/`

## Changelog
Следуем **Keep a Changelog** и **SemVer**. Все изменения добавляются в `[Unreleased]`.

## Секреты и бинарные файлы
- В репозитории — только шаблоны `*.example`.
- Бинарные файлы и крупные артефакты не коммитятся.
